## 4. ระบบการสอบย้อนกลับ (Traceability System)

### 4.1 สอบย้อนกลับจากสินค้าสำเร็จรูปจนถึงแหล่งวัตถุดิบและบรรจุภัณฑ์

ระบบ SAP Traceability Module เชื่อมโยงข้อมูลตั้งแต่:
- **สินค้าสำเร็จรูป** (Finished Good Lot No.) → **รอบการผลิต** (Production Batch) → **วัตถุดิบที่ใช้** (Raw Material Lot) → **ผู้ขาย/แหล่งกำเนิด** (Supplier/Origin)
- **บรรจุภัณฑ์** (Packaging Lot) → **ผู้ผลิตบรรจุภัณฑ์**

### 4.2 สอบย้อนกลับรอบการผลิต

ทุกรอบการผลิตบันทึกข้อมูลดังนี้:
- หมายเลข Batch / Lot Number
- วันที่และเวลาผลิต, กะการผลิต
- วัตถุดิบที่ใช้ (Lot Number ของทุกรายการ)
- ผลการตรวจ CCP (Metal Detector, X-ray)
- ผลตรวจ QC (COA)
- บรรจุภัณฑ์ที่ใช้ (Lot Number)
- รายชื่อลูกค้าที่จัดส่ง, จำนวน, วันที่ส่ง

### 4.3 ความเร็วในการสอบย้อนกลับ

**เป้าหมาย:** สามารถระบุข้อมูล Traceability ครบถ้วนภายใน **4 ชั่วโมง** นับจากได้รับแจ้ง

**ผลการทดสอบ Mock Recall ล่าสุด (ธ.ค. 2568):**
- เวลาที่ใช้: 2 ชั่วโมง 45 นาที
- สามารถระบุ: Lot วัตถุดิบ, รอบการผลิต, ลูกค้าที่จัดส่ง, ปริมาณ — ครบ 100%
- ผลประเมิน: **ผ่าน** (ตามเป้าหมาย ≤ 4 ชั่วโมง)

## 5. ระดับการเรียกคืน

| ระดับ | สถานการณ์ | การดำเนินการ |
|-------|----------|-------------|
| Class I | มีความเสี่ยงต่อสุขภาพ/ชีวิตผู้บริโภค | เรียกคืนทันที + แจ้ง อย./หน่วยงานราชการ |
| Class II | อาจก่อให้เกิดอันตรายชั่วคราว | เรียกคืนภายใน 48 ชม. + แจ้งลูกค้า |
| Class III | ไม่มีความเสี่ยงต่อสุขภาพ แต่ไม่เป็นไปตามข้อกำหนด | Withdrawal จากตลาด + แจ้งลูกค้า |

## 6. ขั้นตอนการเรียกคืน

1. **รับแจ้ง** — จากลูกค้า, ผลตรวจ QC, หน่วยงานราชการ, ผู้บริโภค
2. **ประเมินความเสี่ยง** — ทีม QA ประเมินภายใน 2 ชั่วโมง กำหนดระดับการเรียกคืน
3. **ตัดสินใจ** — ผจก.ทั่วไปอนุมัติการเรียกคืน
4. **ระบุขอบเขต** — Traceability: Lot ที่เกี่ยวข้อง, ปริมาณ, ลูกค้าที่ได้รับ
5. **แจ้งลูกค้า** — โทรศัพท์ + อีเมล + หนังสือแจ้งอย่างเป็นทางการ
6. **เรียกคืน** — จัดส่งรถรับสินค้าคืน หรือให้ลูกค้าส่งกลับ
7. **กักกันและจัดการ** — สินค้าที่เรียกคืนกักกัน → ตรวจสอบ → ทำลาย/Reprocess
8. **สอบสวนสาเหตุ** — Root Cause Analysis (RCA)
9. **แก้ไขป้องกัน** — Corrective & Preventive Action (CAPA)
10. **ปิด Case** — รายงานสรุป → Management Review

## 7. การทดสอบ Mock Recall

- ดำเนินการปีละ 2 ครั้ง (มิ.ย. และ ธ.ค.)
- ทดสอบทั้ง Forward Traceability (วัตถุดิบ → สินค้า → ลูกค้า) และ Backward Traceability (ลูกค้า → สินค้า → วัตถุดิบ)
- เป้าหมาย: ดำเนินการเสร็จภายใน 4 ชั่วโมง, ข้อมูลครบถ้วน ≥ 98%
